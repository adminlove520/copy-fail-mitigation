# CVE-2026-31431 (Copy Fail) 漏洞缓解工具包

这是一个针对 Linux 内核本地提权漏洞 **CVE-2026-31431 (Copy Fail)** 的检测与缓解工具包。

## 漏洞说明

- **漏洞编号**: CVE-2026-31431
- **别名**: Copy Fail
- **危害等级**: 高危 (CVSS 7.8)
- **影响范围**: Linux 内核 4.14 及以上版本（直至修复版本 6.18.22, 6.19.12, 7.0）。
- **描述**: 由于 `algif_aead` 模块中的不当优化，攻击者可利用 AF_ALG 接口非法改写文件页缓存（Page Cache），从而实现本地提权。

## 工具列表

1. `check_cve_2026_31431.sh`: 深度检测脚本。
   - 支持 **i18n** (自动识别中英文)。
   - 检查内核版本。
   - 识别发行版补丁（Ubuntu/RHEL等）。
   - 检测 `algif_aead` 模块加载状态及活跃用户。
2. `fix_cve_2026_31431.sh`: 修复/缓解脚本。
   - 支持 **i18n** (自动识别中英文)。
   - **Apply**: 禁用受影响模块（推荐）。
   - **Rollback**: 恢复系统设置。
   - 支持非交互模式 (`-y`)。

## 使用方法

### 1. 环境准备
确保脚本具有执行权限：
```bash
chmod +x check_cve_2026_31431.sh fix_cve_2026_31431.sh
```

### 2. 系统检测
```bash
./check_cve_2026_31431.sh
```

### 3. 应用缓解措施 (推荐)
通过禁用内核模块来阻断攻击路径。脚本会自动检查系统影响：
```bash
sudo ./fix_cve_2026_31431.sh apply
# 或者使用简写（默认为 apply）:
sudo ./fix_cve_2026_31431.sh -y
```

### 4. 恢复设置
如果发现业务受影响，可以随时撤销：
```bash
sudo ./fix_cve_2026_31431.sh rollback
```

## 支持系统
- **架构**: x86_64, aarch64 (ARM64)
- **发行版**: Ubuntu, CentOS, RHEL, Debian, SUSE
- **信创系统**: 麒麟 (Kylin), 统信 (UOS), 欧拉 (openEuler) 等。

## 免责声明
本工具仅供安全应急使用。禁用 `algif_aead` 可能会影响极少数直接调用内核加密接口的应用。请在生产环境应用前进行充分测试。
